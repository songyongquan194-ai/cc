# 更新日志（CHANGELOG）

本项目遵循语义化版本。日期格式 YYYY-MM-DD。

## [0.1.0] - 2026-05-30

首个完整版本：Windows C 盘安全清理与文件冷藏工具。**确定安全的自动清理；不确定的解释清楚、分类展示、优先迁移到备份盘冷藏；高风险内容默认不处理。**

技术栈：Electron 33 + React 18 + TypeScript 5 + Ant Design 5 + electron-vite；存储 sql.js（WASM SQLite，零原生编译）。单测 **142/142** 通过，node/web typecheck 全绿。

### 核心能力（M0–M8）

- **安全内核**：`SafetyGuard` 八步校验（盘根/通配/禁止目录/排除目录/符号链接/占用），禁止目录硬规则优先于一切规则与 AI。
- **扫描与分类**：快速/深度扫描，按风险等级与类别分类展示，流式进度、可取消；附"为什么"解释。
- **安全清理**：仅清理 `safe/low` 且 `action=clean` 项，逐项二次确认，可清空回收站，全程留痕。
- **迁移与冷藏**：`copy(.part)→verify(sha256)→commit→delete_source` 状态机；源仅在校验通过且提交后才删；迁移前展示「释放 C 盘 / 占用备份盘 / 迁后剩余」三数字；`manifest.json` 与 `cold_items` 双写。
- **恢复**：冲突处置（保留两者/覆盖/取消）、自动重建父目录、空间检查；恢复到系统关键目录硬阻止；失败绝不删冷藏副本。
- **引导与首页**：首次引导（数据处理说明 + 备份盘设置）、首页概览、操作记录（导出 JSON/CSV）。
- **重复文件候选**：仅展示、不默认删除。
- **观察列表**：对"暂不决定"的文件记录并定期提醒，绝不移动/删除文件。
- **AI 顾问（可选，本地·零云端）**：连接本机 Ollama 兼容端点（默认 `http://localhost:11434`）；按需"为什么"解释；自然语言→清理规则（预览确认后才以最低优先级生效）；仅建议、不拥有删除/迁移权。
- **可选托盘**：默认关闭、关窗即退出、不常驻；启用后最小化到托盘并对到期项发本地通知，不联网、不后台监控。
- **打包**：electron-builder NSIS 安装包 + 免安装 `win-unpacked`，自带应用图标。

### M9 真机加固与增强

- **清理性能**：有界并发删除池（默认 16）+ 删除路径跳过占用预检 + 节流聚合进度 + 实时进度条，解决"清理慢/界面像卡住"。
- **扫描会话常驻**：扫描状态提升到常驻 store，**切换界面不再中断扫描**；中断改为显式「停止扫描」二次确认。
- **迁移兜底排除运行库**：深度扫描"大文件兜底迁移"排除可执行/动态库/模型/运行时组件（`.dll/.exe/.model/.onnx`、`ComponentStore` 等），避免迁走应用运行库导致软件报错。
- **源已消失/占用判跳过**：新增错误码 `E_SOURCE_GONE`；清理与迁移把"源已不存在/正在使用"记为**跳过而非失败**，迁移报告新增「跳过」列。
- **冷藏区可读性**：原路径悬停看完整路径 + 一键复制 + 备份位置；新增搜索（原路径/备份位置/分类）。
- **AI 文件识别**：扫描结果与冷藏区可「AI 识别本页」，一句话判断"这大概是什么"。**仅对安全的小文本文件**读取开头片段提升准确度，敏感/高风险/二进制只看元数据；内容只发本机模型、绝不出网；模型不可用降级为按路径的启发式描述。
- **AI 检测修复**：探活去掉 GET 的空 body（修复始终"不可用"）；检测改为纯连通性测试。

### 安全与隐私不变量

- 所有扫描与分析在本机完成，**不向云端上传文件内容或路径**。
- AI 只连本机模型；默认仅元数据，识别读片段也只发本机、绝不出网；高风险/禁止/敏感项一律不读内容。
- 写操作目标经 `SafetyGuard` 校验：禁止写入系统关键目录、盘根；备份盘不能是系统盘。
